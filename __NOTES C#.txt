--- Visual Studio ---

    Общее:
F1                       -- Контекстная ссылка на документацию
F12                      -- Метаданные метода
Ctrl + Space             -- Контекстная подсказка
Ctrl + Alt + J           -- Object Browser (with search)
Ctrl + R, Ctrl + R       -- Переименовать все

    Отладка:
F5                       -- Запуск программы под отладчиком / продолжение после точки останова
F10                      -- Шаг без захода внутрь метода (Step Over)
F11                      -- Шаг с заходом внутрь метода (Step Into)
Shift + F11              -- Выполнение инструкций вплоть до выхода из текущего метода. (Step Out)

--- ТЕРМИНОЛОГИЯ ---

Кодовый файл             -- Один из файлов на языке C#.
Проект                   -- Совокупность кодовых файлов, которые могут быть скомпилированы в сборку: программу или библиотеку.
Сборка                   -- Результат компиляции проекта. Как правило это *.exe или *.dll файл, содержащий инструкции для компьютера.
Решение (solution)       -- Несколько проектов, объединенные общими библиотеками и задачами. Как правило открывать с помощью Visual Studio нужно именно файл решения (.sln), хотя можно открыть и отдельный проект (.csproj файл). Имейте в виду, если открыть отдельный кодовый файл, не открывая проект или решение, то не будет возможности его запустить. Это распространённая ошибка новичков.
Reference                -- ссылка внутри проекта на другие сборки. Сославшись на другую сборку можно будет использовать код из неё.
Метод                    -- Последовательность действий. Аналог функций, процедур и подпрограмм в других языках. В устной речи часто используют все эти слова как синонимы, но в спецификации на язык C# используется термин «метод».
Класс                    -- Совокупность данных и методов. Все сборки состоят из скомпилированных классов.
Пространство имен        -- Совокупность классов, логически связанных между собой. Между сборками и пространствами имен нет прямого соответствия: в сборке может хранится несколько пространств имен, а разные классы одного пространства имен могут быть определены в разных сборках. После успешной компиляции, в директории проекта создается поддиректория bin/Debug, в которой и оказывается сборка — результат компиляции — exe или dll файлы вашей программы.
Рефакторинг              -- Переделка кода в целях улучшения читаемости
